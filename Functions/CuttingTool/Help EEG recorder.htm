Very short introduction

This computer program consists of two modules. One for the experimenter only (the stimulus designer and the other for the test subject (the stimulus presenter). The stimulus designer while the second module is used to run the actual experiment (the stimulus presenter). The program consists of a series of timers that allow you to automate (1) the presentation of stimuli (2) the collection of behaviour responses to your stimuli by the test subject and (3) to generate programmable digital output.

The digital output of this program can be used as digital input during EEG measurements while the stimulus presenter provides the sensory or behavioural input for the EEG experiment.

The stimulus designer also contains a number of tools that might be helpful during the preparation of your experiment.

The stimulus presenter (exists in white and black)

 

 

Basic functionality and designing an experiment

 

Basic timer functionality:

Start delay: Time until first timer event from the moment the timer is started (e.g. the time between the start of the ImageTimer and the actual presentation of the first image).

Period: Time between two timer events (e.g. The time between onset of image1 and onset of image2).

Execution: The amount of timer events (e.g. number of images to be presented).

 

Starting timers

Main timer: Starts when the Start button on Stimulus presenter is pressed.

 

All other timers are started by each main timer event.

If you use more than one main timer event this implies that each full set of operations executed by one of the sub-timers should be finished in a time span that is shorter than the period of the main timer.

 

Visualizing your experimental design using Plot first cycle:

To check whether the order and timing of events will be executed as you intended you can plot the experimental design using the Plot first cycle button.

Before you plot:

-Set Period and Executions main timer to 1 .

-Set Signal end of the main timer to the time that you would like to use as the period of your main timer (or the actual end if you only plan to use one main timer event).

 

The plot displays all timer events in the first cycle of the main timer.

There are two subplots:

1)    The stimulus events

2)    The user input events

 

The axis are linked, use zoom to visualize details.

 

Use DIO

If marked the stimulus presenter will generate digital output through the NI USB-6501

 

Start the experiment

When Start on Stimulus designer is pressed:

1)    Some stimulus data is preloaded in memory

2)    Hardware is started up

3)    Stimulus presenter is loaded

 

Stimulus presenter:

When Start on Stimulus presenter is pressed:

1)    The Main timer is started

2)    The program adjusts itself to the screen (less than 1 sec, use at least 1 sec start delay on main timer)

 

Terminating execution while the program is running:

Go to Stimulus designer and press clear all > close Stimulus presenter.

 

Saving or continuing:

After the experiment you can:

1)    Save your results to a matlab struct, (and delete timers and data from memory)

2)    Continue to add data to the current struct while timers and counters are removed from memory

 

3)    Delete all data, timers, variables and counters.

Tools

The stimulus designer provides a number of programs to create or manipulate stimuli. They can be found under Tools in the taskbar.

 

-Text editor

Enables you to open a *.txt file and adjust the content.

- Image editor

Enables you to crop a picture and get info about the picture size and pixel information

- Audio recorder

Under duration simply enter how long you want the audio file to be. Press record and it starts recording. Afterwards you can play it back, and if necessary re-record it, otherwise save it as a *.mat file which can be read by the stimulus designer program.

- Advanced audio tools

This tool allows more elaborate manipulation of existing sound files (such as cutting a file, adding two files together, creating stereo audio streams, or generate virtually moving sound sources) It has its own help documentation.

 

- Video recorder

The video recorder allows you to make video files using the laptops webcam. It has a number of different function and its own help documentation. Here follows a short instruction to make a simple movie:

1.    Open the video recorder, a new window pops up.

2.    Select the hardware and its settings in the Hardware Browser (if the Hardware Browser does not open automatically select it under Desktop on the taskbar).

3.    Select the LenovoEasyCamera with the maximal resolution of 640x480.

4.    In the Frames Per Trigger select the number of frames you want the movie to be, or set it to infinite in order to manually stop the recording when you are done.

 

5.    Go to the Logging tab, name the file (Filename) and its location (Log to: Disk).

6.    The resulting movie will be save as a *.avi file in the directory where Matlab is located.

 

 

7.    Start and stop recording with the Start Acquisition and Stop Acquisition buttons respectively. Before you start recording you can check what the resulting image will look like using the Start Preview button.

 

- Video editor

The video editor allows you to load a movie and to cut, crop, rotate movies or adjust the color. The interface is pretty straightforward and intuitive.

 

- Trial randomizer

The trial randomizer tool is useful for experiments where you have a few images and/or sounds (up to five) that you want to present repeatedly in a random order.

1)    Fill in the total number of trials you want, and how many conditions

2)    Choose how you want to determine the trial order

a.    Read trial order from an excel file you have created yourself. The file needs to consist of just one column of condition numbers in the order you want them to appear (e.g. 1,3,4,2)

b.    Let the tool generate a random order

                                                                          i.    Indicate how many trials you want per condition, total should be equal to the number you filled in under step 1. The number of trials doesnt need to be equal for each condition (e.g. condition 1 = 90, condition 2 = 10)

                                                                        ii.    Shuffle will generate a random order of trials, including all conditions

                                                                       iii.    Plot conditions will plot what condition each trial is, and a histogram of the number of trial per condition.

                                                                       iv.    Export to excel lets you save the generated trial order to an excel file which you will need when splitting the behavioural and EEG data according to condition.

3)    Indicate which files you want the tool to read Note, the template files should be placed in the current folder of Matlab or in the Stimuli_Folder.

a.    Check the boxes if you want to read image and/or audio files.

b.    Type in the name of the template file. For the audio file also choose if the template is .wav or .mat

4)    Select to which stimulus timer the file should be written (1 or 2)

5)    Click read & write stimuli and select the folder where you want to write to. A message will appear if all files are successfully read and written.

 

 

 

Timer Descriptions

Main timer

The main timer is essential to the experiment design and should always be on (by checking the box). Each execution of this timer starts all other selected timers.

 

Start delay: Time between the start of the experiment and the first event of the main timer.

Period: Time between events of the main timer. This only applies if the number of executions is larger than 1, otherwise this can be set to 1.

Executions: The number of main timer events (i.e. the number repeated cycles of all other selected timers)

Signal end: The timing of the end of the experiment, this should be a minimum of the start delay plus the period times the number of executions (start delay + (period x executions))

TTL on: when the user presses the start button. (see DIO)

 

Every event generated by the main timer triggers the start of all other timers. The start delay of all other timers is relative to the main timer event, and the number of executions applies to one period of the main timer.

 

Stimulus Timers

Image timer:

Time visible: The time that the image is visible.

Images should be placed in the current folder of matlab.

Images are not preloaded. After each timer execution the images are red from hdd and then plotted. Timing is thus not highly accurate but still pretty good.

Images should be *.jpg

Image names should be:

1im1, 2im1,.. (n)im1 for timer 1

1im2, 2im2,.. (n)im2 for timer 2

Images are stretched to fit the axis in Stimulus_presenter

For decent images:

Fit the aspect ratio to the axis

Or fit, in the GUI edit mode, the axis to the image

Images can downloaded from the internet, or make them yourself in Paint or PowerPoint or with your phone.

When you make figures in PowerPoint, select the figure, right click and select Save as Picture, and then save it as the default file type (Enhanced Windows Metafile), then open the saved file in paint and save it again, but this time as *.jpeg.

 

Sequence timer

Each timer can present only one image sequence.

The images are loaded in memory before the onset of the experiment. This can consume a substantial amount of memory!!!!

Timing of the first image should be pretty accurate.

Visible: This is the time a frame is visible. This time is not accurate! It is however true that the smaller the number the faster the sequence.

The actual frame rate is measured during the presentation and it is printed in the command window of matlab.

The image sequence should be in the following designated folder:

seqence1 for sequence timer 1

seqence2 for sequence timer 2

The folders are already present in the program folder.

Images should be *.jpg

Image names should be:

000.jpg, 001.jpg, 002.jpg(n).jpg

Images are stretched to fit the axis in Stimulus_presenter

 

 

Movie timer

Plays a sequence of movies.

A sequence of movies can only consist of *.avi or *.mpeg files. You cannot mix formats within a timer but you can mix between timers.

All movie files should be placed in the current folder of matlab.

Movie names should be :

vid1_1, vid1_2,.. vid1_(n) for timer 1

vid2_1, vid2_2,.. vid2_(n) for timer 2

 

All movies in the sequence are preloaded in memory before the onset of the experiment. This can consume a considerable amount of memory!!!!

Timing of playback should be pretty accurate.

Movies are NOT stretched to fit the axis. They are played back in their natural aspect ratio. It is not possible to stretch movies to a full screen format in matlab (except with an obscure function in the robotics vision toolbox that we dont have).

Screen location: The center of the movie frame is fitted to the center of the axis in stimulus_presenter.

The frame rate is read from the *.avi or *.mpeg file. To adjust the frame rate you thus have to adjust it in these files.

Remember that you can find a movie editor and a video recorder under tools in the stimulus_designer.

The timer does not stop the movie, it will play till the end.

 

Audio Timer

Plays a sequence of audio files.

Audio files can be *.wav files or *.mat files (matlabs data matrix format).

All audio files should be placed in the current folder of matlab.

*.mat files can be created using the audio recorder under tools in the stimulus presenter.

Within a timer *.mat and *.wav formats can be mixed.

Audio file names should be :

1au1.mat, 2au1.wav, 3au1.mat, 4au1.wav,. (n)au1.wav for timer 1

1au2.mat, 2au2.wav, 3au2.mat, 4au2.wav,. (n)au2.wav for timer 2

Audio files (*.wav and *.mat) can be adjusted with the advanced audio toolbox under tools in the stimulus presenter.

The timer cannot stop playback, you will have to face the music until it finishes.

 

Text timer

This function reads text from a text file.

The text file should be present in the current folder of matlab.

The text file should have the following name:

Text1.txt for timer 1

Text2.txt for timer 2

Under tools in stimulus_designer is a simple text editor available but notepad will do the job just as well.

Example of six different texts that will be presented at each instance of the text timer:

Something

Something2

Something else

And something different again

02040404

2034 kaDn 103i

The text files are loaded before the onset of the experiment.

Timing of the text presenter should be very accurate.

 

External application timer

This function plays back a playlist in the applications that are the default applications for the specified file format.

All files in the playlist should be present in the current folder of matlab.

Make sure to add the file format extension (*.mp3, *.flv etc) to the file name in the play-list.

Use this function for example to:

Play full screen movies

Play movie or audio formats not supported by matlab

Start an automated PowerPoint

Start a website

Start your own program

Etc

Stimulus_presenter has two behavioral options while the external application is running:

1)    It can move to the windows taskbar and pop back up after the external app .

To move to task bar:

Indicate in Back to main the amount of seconds you want the program to stay in the taskbar after timer execution.

 

2)    It can stay put

 

To stay put:

Set Back to main to 0.

Timing: Timing is not very accurate. Nothing is preloaded in memory.

Some tips and tricks:

-You can improve timing by preloading the applications you want to use in windows. This saves on start-up time.

-You can change the default application, windows uses, to open particular files.

-You can set movie players like vlc to always playback in full screen mode.

Whereas it is possible to start external applications from matlab it is not possible to close them! So all applications will stay open, this can drain system resources.

 

 

 

Feedback timers

With the feedback timers you can ask for input from the subject in the form of a mouse click, hitting a keyboard button, or typing in a reply. The functions of the different timers will be described in more detail below.

 

 

Text input timer

At each execution of this timer:

-Text-input box 1 will become visible

-Focus will be given to this box (so people can immediately type, no moussing required)

-This box disappears when:

-The user presses the enter key

-When the amount of seconds in the time visible field elapsed

Data: All text that the subject entered is saved. After the experiment the Text_input_1 and 2 fields in the saved data struct contain this input as a cell array.

There is no limit to the amount of text that can be entered per timer execution.

Time is not measured.

 

Mouse input timer

Upon execution of this timer the program:

-Starts a timer

-Waits for a mouse click

If you dont click the program will wait forever! Make sure the subject clicks at some point.

At a mouse click (the down stroke):

-The time form onset of the timer until the click is very accurately measured

-The X and Y coordinates of the mouse on the screen is recorded.

Data: After the experiment the Mouse_reaction_time_1 and 2 and the Mouse_XY_1 and 2 fields in the saved data struct contain the results (in arrays).

 

Key reaction timer

Upon execution of this timer the program:

-Starts a timer

-Waits for any key strike

At a key strike:

-The time from onset of the timer until the strike is very accurately measured

-The struck key is saved

When no key is pressed before the timer end

-No time is saved but the a NaN is saved (stands for Not a Number)

-The sign ~ is saved instead of a key

All keys on the keyboard can be used

No more than one key per timer event can be used

Data: After the experiment the Keyboard_reaction_time_1 and 2 and Keyboard_reaction_key_1 and 2 fields in the saved data struct contain the results (in arrays).

 

 

Experimental results

During the experiment the program records all user input. At the end of the experiment you can choose to clear all responses, keep them in memory so you can add more data during the nest experiment, or save data to disk.

To view results:

-Save the responses to the disk (as a *.mat file in the current folder of matlab, in the example below test_output_01).

To view, first load the files to Matlab workspace:

Double click (you wont see anything happening yet). Then there are two ways you can view the user responses :

1) by typing UsIp in the command window. This will show the variables in the struct UsIp in the command window.

2) Go to workspace (if not visible look under the Desktop button in the taskbar) and double click on UsIp. Subsequently you can double click the data tables. In the example below keyboard responses were recorded. The timing and the key-strokes were thus saved (here z and ?).

After double clicking UsIp, a new window containing the data tables opens (e.g. keyboard, mouse, text). Double click on the table to view the data. You can copy & paste information to Excel or SPSS for further analysis.

What is a struct?

A struct is a virtual box that programmers use to store data that belong together under one file name. In this experiment the results are saved in a struct (your_name.mat). If you import the struct (right mouse-click, import) you can see all your data tables in the matlab workspace. Double-click, or type a tables name in the command line, to see the data.

 

 

Digital output (TTL)

In Stimulus designer you can configure digital output that will be produced by Stimulus presenter during your behavioural experiment. This digital output can be passed on to the digital input channels of the EEG data-acquisition hardware.

 

The EEG recorder has 8 digital input channels

The DIO attached to the stimulus presenter has 24 digital output channels.

So, depending on your experiment you should choose which output channels you connect to the digital input of the EEG recorder.

Output voltage varies between 4.5 and 5 V

 

Configuring digital output

Each timer can be configured to generate digital output by signing the box TTL on below the timer.

If you use digital output you also have to sign the box Use DIO in the Check & Run panel.

 

 

DIO pinout

 

Ni USB-6501

 

Each timer has been assigned to one digital output channel according to the map below:

 

To help those that are really annoyed by this apparent chaos here is the rationale:

Ignore the clear numbers (1 through 32).

The digital output channels are the channels indicated with a P-number.

The timers appear in the same order as they appear on the stimulus designer on subsequent P-numbers.

 

Connecting the DIObox to the MOBIlab

Each timer of the Stimulus_designer sends a TTL pulse to a specific port on the DIO box. However, we can record only six digital channels besides the eight EEG channels, so you have to choose which of the ports (TTLs) you want to record. For example, if your Experiment involves Image Timer1, Image Timer 2, Audio Timer1 and Key Reaction Timer1 then you should plug wires into port 17/P0.0 (for the Main Timer), 19/P0.1, 20/P0.2, 24/P0.7 and 15/P2.1.

Note: You should also always plug in a ground electrode (the grey wire) into one of the GND ports!

Which Timer is recorded as which digital channel number (DIO) in the EEG_recorder is determined by the color coding of the wires as follows:

DIO 1 = black

DIO 2 = red

DIO 3 = blue

DIO 4 = pink

DIO 5 = yellow

DIO 6 = green

Ground = grey

 

Connecting the Digital output to the digital input cannels of the EEG recorder

Use a screwdriver to attach the digital input wire to the desired digital output channel.

Example: If you would like to see the start of the experiment (main timer) on your first digital channel in the EEG_recorder, connect the black wire to channel 17/P0.0 of the DIO.

Sample rate

Always 256 on the MOBIlab+

Save to SD

When on data is also logged on internal SD card on MOBIlab+

Make sure you inserted the SD card between the batteries!

Save to RAM / disk / RAM & disk

Here you can choose how the live recorded data is saved: only to RAM (working memory), only to (hard) disk or to both.

When you only save to RAM you will have to save the data to the hard disk after recording (by clicking the save button in the menu).

data = daqread(new_data.daq)

save(filename , data) here filename is the name that you want to give the dataset.

Adjusting live view plots

 

Preview sample length

The number of samples that are plotted in the time domain figure. The larger the number, the more data is plotted. Examples

         Setting preview sample length to 256 samples (= 1 second) then constantly the latest second of data is plotted.

         A preview length of 1024 samples will plot the latest 4 seconds of recorded data.

 

FFT sample length

The number of samples that is used to generate the FFT (Fast Fourier Transformation) and plot the signal in the frequency domain.

         The FFT sample length should always be smaller or equal to the preview sample length. (FFT length <= preview length)

 

Channel spacer

This number determines the distance between the plotted channnels

         Smaller number = bigger signal amplitude

 

FFT frequency range

Set the range of frequencies that are plotted live in the powerspectrum window
